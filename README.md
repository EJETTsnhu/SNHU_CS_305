# SNHU_CS_305
Artemis Financial is a financial services company that needed help improving the security of its software. The company handles sensitive financial data, so protecting that information was very important. My role was to review the application, find security risks, and improve the code to reduce those risks. I completed a vulnerability assessment and made changes to strengthen the system.

During the project, I used automated tools to scan for vulnerabilities and reviewed outdated dependencies in the application. One challenge was understanding which vulnerabilities were serious and which were lower risk, but this helped me learn how to evaluate security issues more carefully. I improved security by updating dependencies, using secure communication protocols like HTTPS, and applying better coding practices. After making changes, I rebuilt the project and ran the scans again to make sure everything still worked and that I did not introduce new problems. This step was important because security updates should not break functionality.

This project shows that I can identify software risks, apply secure coding practices, and verify that improvements are effective. It also demonstrates that I understand why security matters. Strong software security protects customer data, prevents financial loss, and builds trust. I would show this work to employers as proof that I can assess vulnerabilities, improve security, and document my process clearly.
